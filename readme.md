# ModbusUDP Control Server

Асинхронный сервер на Python для мониторинга и управления устройствами через Arduino-шлюз по протоколу UDP/Modbus. Реализует веб-интерфейс, WebSocket-стриминг данных в реальном времени, ПИД-регулятор, логику управления реле и математическую обработку датчиков (ДМРВ, термопары, аналоговые пины).

## Основные возможности
- Асинхронная UDP-связь с Arduino (отправка/приём бинарных пакетов)
- Modbus over UDP (поддержка функций 0x03 Read и 0x06 Write)
- Веб-панель управления (Tornado + HTML/JS + WebSocket)
- Мониторинг в реальном времени: частоты, аналоговые датчики, термопары, состояние реле
- ПИД-регулятор и гистерезисный контроллер реле для автоматизации
- Обработка ДМРВ: конвертация напряжения в массовый расход и температуру
- Гибкая настройка через INI-файл (слэйв-устройства, регистры, порты, коэффициенты)

## Архитектура проекта
| Файл | Назначение |
|------|------------|
| `main.py` | Точка входа. Запуск Tornado HTTP/WS сервера, UDP-слушателя, маршрутизация запросов |
| `devices/arduino_board.py` | Формирование UDP-пакетов, управление Modbus-командами, реле, термопарами |
| `devices/slave_device.py` | Представление Modbus-устройства, парсинг регистров и аналоговых пинов из конфига |
| `devices/pid.py` | Реализация ПИД-регулятора и гистерезисного контроллера реле |
| `devices/dmrv.py` | Математическая модель ДМРВ (расчёт расхода и температуры по напряжению) |
| `utils/answer_parsing.py` | Декодирование бинарного ответа от Arduino в словарь Python |
| `utils/modbus_request.py` | Dataclass для валидации и сериализации Modbus-запросов |
| `templates/index.html` | Веб-интерфейс с формами управления и живым логом |
